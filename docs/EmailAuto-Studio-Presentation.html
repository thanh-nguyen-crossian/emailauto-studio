Overview

What it does

The problem

Hand-built emails, slowly

The team manually wrote copy, hand-built HTML per brand/tier/segment, then ran a cleanup script before loading templates into SendGrid — repetitive and error-prone.

The solution

One guided studio

A campaign brief produces every tier × segment variant with proven, on-brand copy and SendGrid-ready HTML — previewed, QA'd, saved, and synced in a few clicks.

Built on proof

The winning playbook

Prompts encode the analysis of 46 real campaigns: brand personas, the Hook Contract, and win/fail guardrails — so output follows what actually converts.

Closes the loop

Straight into SendGrid

Push as a Design, or run the team's cleanup pass and create a Dynamic Template — returning the d-… Template ID for sends.

Security

Verified on production

  • Authenticated paid routes
    generation + SendGrid sync require an active user (401 otherwise)
  • Admin authorization
    token + is_admin + active; non-admin → 403
  • Row-Level Security
    users see only their own data; no self-grant of admin
  • No secret leakage
    service-role / SendGrid / Anthropic keys are server-only
  • Sandboxed preview
    email rendered in a no-script iframe; inputs escaped
  • HTTPS + HSTS & security headers
    X-Frame-Options, nosniff, Referrer/Permissions-Policy

Recommended next: rotate shared keys · rate-limit generation · optional CSP & signup domain allowlist.
